Algebraic Modeling

Samuel Burer

Last updated: January 20, 2017

What is Algebraic Modeling?

Algebraic modeling of optimization problems uses algebraic expressions to specify the variables, constraints, and objective function in a manner that clearly separates the logical model from the data

  • logical model = high-level mathematical relationships between variables, constraints, and objective function

  • data = the numbers/parameters that connect the logical model to the real-world

Concept: Index Sets

An index set (or just set) is a collection of objects over which variables, constraints, and data/numbers/parameters can be defined. E.g.:

  • You make three phone sizes: small, medium, and large

  • The corresponding set is the collection of words/strings {small, medium, large}

  • Or {s, m, l}

Concept: Index Sets (cont’d)

E.g.:

  • You are planning for the months of July, August, September, and October

  • {July, August, September, October}…

  • Or {Jul, Aug, Sep, Oct}

  • Or {7, 8, 9, 10}

Concept: Index Over a Set

  • Let \(S\) represent a set

  • An index over \(S\) is a symbol used to represent looping over the elements of \(S\)

  • Usually denote as a letter such as \(i\) or \(j\)

E.g.:

  • \(i\) loops over {small, medium, large}
  • \(j\) loops over {7, 8, 9, 10}

Concept: Index Over a Set (cont’d)

We also sometimes write:

  • \(i \in\) {small, medium, large}
  • \(j \in\) {7, 8, 9, 10}

The mathematical symbol \(\in\) means “in”

Concept: Index Over a Set (cont’d)

We can define variables over a set:

  • \(x_i\) for all \(i \in\) {small, medium, large}
  • \(y_j\) for all \(j \in\) {7, 8, 9, 10}

Or data/numbers/parameters:

  • profit \(p_i\) for all \(i \in\) {small, medium, large}
  • demand \(d_j\) for all \(j \in\) {7, 8, 9, 10}

Or constraints, etc.

Concept: Summation

  • Let \(S\) represent a set

  • A summation over a set is a loop that multiplies variables with data and adds everything up

  • Mathematically, a linear combination of variables with data as the coefficients

  • Similar to Excel’s SUMPRODUCT command

E.g., with set \(S\), index \(i\), variables \(x_i\), and data \(a_i\): \[\sum_{i \in S} a_i x_i\]

Concept: Summation (cont’d)

E.g.:

\[\sum_{i \in {\tt \{s, m, l\}}} p_i x_i = p_{\tt s} x_{\tt s} + p_{\tt m} x_{\tt m} + p_{\tt l} x_{\tt l}\]

\[\sum_{j \in {\tt \{7, 8, 9, 10\}}} d_j y_j = d_7 y_7 + d_8 y_8 + d_9 y_9 + d_{10} y_{10}\]

Concept: Summation (cont’d)

  • Summations can be included in constraints and in the objective function

  • Are convenient because:

    • they are compact

    • they help us keep the logical model separate from the data/numbers/parameters

Double Sets and Summations

  • Let \(S\) and \(T\) be two sets

  • We often consider the Cartesian product of \(S\) and \(T\), which is denoted as \(S \times T\)

  • \(S \times T\) consists of all pairs \((i,j)\), where \(i \in S\) and \(j \in T\)

Double Sets and Summations (cont’d)

E.g.:

{ (small, 7), (small, 8), (small, 9), (small, 10), (medium, 7), (medium, 8), (medium, 9), (medium, 10), (large, 7), (large, 8), (large, 9), (large, 10) }

Double Sets and Summations (cont’d)

  • Variables, constraints, and data can all be defined relative to \(S \times T\)

  • Summations as well, e.g.: \[\sum_{i \in S} \sum_{j \in T} a_{ij} x_{ij}\]

  • All of this can be extended to triple sets, quadruple sets, etc.
