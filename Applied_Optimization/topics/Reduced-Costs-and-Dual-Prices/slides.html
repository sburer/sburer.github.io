Reduced Costs and Dual Prices

Samuel Burer

Last updated: January 20, 2017

Variables: Bounds

In a LP, each variable \(x\) has a lower bound \(L\) and an upper bound \(U\): \[L \le x \le U\]

Special cases:

  • If \((L,U) = (0,+\infty)\), then \(x\) is nonnegative
  • If \((L,U) = (-\infty,+\infty)\), then \(x\) is free

Note: Bounds \(-\infty\) and \(\infty\) are usually implicit

Variables: Objective Coefficient

When \(x\) appears within the term \(c \cdot x\) in the objective, the value \(c\) is called the objective coefficient for \(x\)

  • If \(x\) does not appear in the objective explicitly, it just means that \(c = 0\)

Variables: At Optimality

Suppose you solve the LP

  • If \(x^* = U\), then, in a sense, the LP would like more of \(x\) in order to further optimize the objective

  • If \(x^* = L\), then the LP would like less of \(x\)

  • If \(L < x^* < U\), then the LP is content having \(x\) in the middle

Variables: At Optimality (cont’d)

The reduced cost of \(x\) is…

  • When \(x^* = U\), the (smallest) change in \(c\) that will induce the LP to start decreasing \(x^*\)

  • When \(x^* = L\), the (smallest) change in \(c\) that will induce the LP to start increasing \(x^*\)

  • When \(L < x^* < U\), the reduced cost is 0, meaning that the LP is already content with \(x^*\)

Variables: At Optimality (cont’d)

  • LP software can report the RC after solving

  • Do not worry about the sign (positive or negative) of the RC; just interpret it ``in the right direction’’

  • RC’s are interpreted one at a time (ceteris paribus) and work ``locally’’

Example 1: Reduced Costs

max   x + 2*y
s.t.  1 <= x + y <= 2
      0 <= x <= 1.5
      0 <= y <= 1.5
  • Optimal solution is \((x^*,y^*) = (0.5, 1.5)\) with value 3.5
  • RC for \(x\) is 0
  • RC for \(y\) is 1

Example 1: Reduced Costs (cont’d)

  • The objective coefficient for \(y\) is \(c = 2\)

  • Since \(y\) has hit its upper bound at \(y^* = 1.5\), the LP is essentially asking for more of \(y\)

  • But if \(c\) were reduced from \(2\) to \(1\) (corresponding to a RC of 1), then \(y\) would no longer be so valuable

  • If \(c\) were reduced further (below 1), then the LP would want to decrease \(y^*\)

Example 1: Reduced Costs (cont’d)

  • For \(x\), the objective coefficient is \(c = 1\)

  • Since \(x\) is between at \(x^* = 0.5\), no change in the objective coefficient is required to make the LP content with \(x\), i.e., the RC is 0

Example 2: Reduced Costs

max   -3*x + 2*y
s.t.  (same constraints)
  • Opt soln is \((x^*,y^*) = (0.0, 1.5)\) with value 3.0
  • RC for \(x\) is -3
  • RC for \(y\) is 2

Example 2: Reduced Costs (cont’d)

  • LP wants less of \(x\)

  • But if \(x\)’s objective coefficient were larger than \(-3 + 3 = 0\), then the LP would actually want more of \(x\)

  • Ceteris paribus the LP would decrease \(y\) if its objective coefficient fell below 0

Constraints: Bounds

Each constraint has a lower bound \(L\) and an upper bound \(U\), e.g.: \[L \le a \cdot x + b \cdot y \le U\]

  • The quantity between the bounds, e.g., \(a \cdot x + b \cdot y\), is called the body of the constraint

  • Let’s use the letter \(B\) to indicate the body

Constraints: Bounds (cont’d)

Special cases:

  • If \(L = -\infty\), then less-than-or-equal-to

  • If \(U = +\infty\), then greater-than-or-equal-to

  • If \(L = U\), then equality

  • If \((L,U) = (-\infty,+\infty)\), then actually not a constraint

Constraints: At Optimality

Suppose you solve the LP…

  • If \(B^* = U\), then, in a sense, the LP would like to increase \(U\)

  • If \(B^* = L\), then the LP would like to decrease \(L\)

  • If \(L < B^* < U\), then the LP has no incentive to change \(L\) or \(U\)

Constraints: At Optimality (cont’d)

The dual price of the constraint is…

  • When \(B^* = U\), the predicted change in the optimal value if \(U\) is increased to \(U + 1\)

  • When \(B^* = L\), the predicted change in the optimal value if \(L\) is increased to \(L + 1\)

  • When \(L < B^* < U\), the dual price is 0, meaning that changes to \(L\) or \(U\) are not predicted to have an impact on the optimal value

Constraints: At Optimality (cont’d)

  • For an equality constraint, i.e., \(L = B^* = U\), the DP is the predicted change in the objective when \((L,U) \to (L+1,U+1)\)

Constraints: At Optimality (cont’d)

  • LP software can report the DP’s after solving

  • DP’s are interpreted one at a time (ceteris paribus) and have only a local effect

  • To be precise: DP’s should be interpreted as predicted rates of change

Example 3: Dual Prices

max   x + 2*y
s.t.  1 <= x + y <= 2
      0 <= x <= 1.5
      0 <= y <= 1.5
  • Optimal solution is \((x^*,y^*) = (0.5, 1.5)\) with value 3.5
  • \(B^* = 2\) and DP is 1

Example 3: Dual Prices (cont’d)

  • If constraint is changed to \(1 \le x + y \le 3\), then the opt val changes from 3.5 to 4.5

  • But keep in mind this is a local effect, e.g., if you keep increasing \(U\), eventually the effect of the dual price will break down

  • Also, if \(U\) is changed to \(2 - 0.5 = 1.5\), then the opt val will change from 3.5 to 3.0

Example 4: Dual Prices

min   (same objective)
s.t.  (same constraints)
  • Opt soln is \((x^*,y^*) = (0.5, 1.5)\) with opt val 1
  • \(B^* = 1\) and dual price is 1
  • Changing to \(0 \le x + y \le 2\) gives opt val 0

Caveat

  • Keep in mind

    • Reduced costs are changes that will cause the LP to want to change its optimal solution

    • Dual prices are predicted changes in the opt val

  • However, if the LP is ``overly constrained,’’ the LP may not be able to realize the effects predicted by the reduced costs and dual prices

    • How do you tell if the LP is overly constrained? If, at optimality, lots of variables and constraint bodies hit their lower and upper bounds
